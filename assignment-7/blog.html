Q :- 1. Difference between local and session storage ?

Session Storage is similar to localStorage; the difference is that while data in localStorage doesn't expire, data in sessionStorage is cleared when the page session ends. Whenever a document is loaded in a particular tab in the browser, a unique page session gets created and assigned to that particular tab.

Q :- 2. Difference between global and block scope ?

In a browser environment, the global scope is controlled by the window object while in Node. js, it's controlled by the global object. Block scopes are what you get when you use if statements, for statements, and the like.

Q :- 4. How many ways to find undefine ?

undefined is a property of the global object. That is, it is a variable in global scope. The initial value of undefined is the primitive value undefined. In all non-legacy browsers, undefined is a non-configurable, non-writable property. (Even when this is not the case, avoid overriding it). A variable that has not been assigned a value is of type undefined. A method or statement also returns undefined if the variable that is being evaluated does not have an assigned value. A function returns undefined if a value was not returned. Anything that lacks a description or is not declared is considered undefined. For example, in computer programming, if a variable is not declared when the program or script is run, you will receive a "no value," "not defined," "unbound," "undefined" error message.
